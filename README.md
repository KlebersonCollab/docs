## 📋 Documentos Disponíveis

### 1. [Guia de Documentação](documentation-guide.md)
Guia completo que explica:
- Diferenças entre User Stories e Use Cases
- Quando usar cada tipo de documentação
- Boas práticas para cada formato
- Ferramentas recomendadas
- Exemplos práticos

### 2. [Template User Story](user-story-template.md)
Template estruturado para criação de histórias de usuário, incluindo:
- Formato padrão "Como/Quero/Para que"
- Critérios de aceite
- Cenários principais e alternativos
- Regras de negócio
- Definição de pronto (DoD)

### 3. [Template Use Case](use-case-template.md)
Template detalhado para casos de uso, incluindo:
- Atores e pré-condições
- Fluxo principal
- Fluxos alternativos e de exceção
- Regras de negócio
- Cenários de teste

### 4. [Template BDD](bdd-template.md)
Template estruturado para documentação BDD (Behavior Driven Development), incluindo:
- Especificação em linguagem Gherkin
- Cenários de sucesso, validação e exceção
- Palavras-chave obrigatórias (Dado, Quando, Então)
- Regras de negócio mapeadas
- Dados de teste e rastreabilidade

### 5. [Exemplo BDD](bdd-example.md)
Exemplo prático de documentação BDD baseado no caso "Cadastrar Anúncio", demonstrando:
- Aplicação real dos conceitos BDD
- Mapeamento de critérios de aceite
- Implementação de testes automatizados
- Rastreabilidade com User Stories e Use Cases

### 6. [Template Plano de Teste](test-plan-template.md)
Template estruturado para criação de planos de teste, incluindo:
- Estratégia de teste e cronograma
- Funcionalidades e requisitos mapeados
- Recursos necessários e riscos
- Métricas de qualidade e relatórios

### 7. [Template Roteiro de Teste](test-script-template.md)
Template detalhado para roteiros de teste, incluindo:
- Cenários de sucesso, validação e exceção
- Casos de teste estruturados em tabelas
- Dados de teste e protótipos
- Execução e rastreabilidade

### 8. [Template Caso de Teste](test-case-template.md)
Template completo para casos de teste individuais, incluindo:
- Passos detalhados e dados de entrada
- Resultados esperados e evidências
- Análise de resultados e defeitos
- Rastreabilidade e aprovações

### 9. [Template PRD](prd-template.md)
Template para Product Requirements Document, incluindo:
- Visão geral do produto e objetivos
- Requisitos funcionais e não funcionais
- Arquitetura e abordagem técnica
- Cronograma e recursos necessários

### 10. [Template TRD](trd-template.md)
Template para Technical Reference Document, incluindo:
- APIs e endpoints detalhados

- Integrações externas e segurança
- Exemplos de uso e troubleshooting

### 11. [Template FRD](frd-template.md)
Template para Functional Requirements Document, incluindo:
- Funcionalidades principais e fluxos

### 12. [Template RFC](rfc-template.md)
Template para Request for Comments, incluindo:

- Cronograma e recursos

### 13. [Template ADR](adr-template.md)
Template para Architectural Decision Record, incluindo:

- Consequências e implementação

### 14. [Template Engineering Guidelines](engineering-guidelines-template.md)
Template para diretrizes de engenharia, incluindo:

- Boas práticas de desenvolvimento
- Processo de desenvolvimento e qualidade
- Ferramentas e tecnologias

### 15. [Template System Design](system-design-template.md)
Template para design de sistema, incluindo:

### 17. [Template C4 Model](c4-model-template.md)

### 18. [Template TRG](trg-template.md)

### 19. [Template Threat Model](threat-model-template.md)

- Análise de riscos e plano de mitigação

### 20. [Template Data Governance](data-governance-template.md)
Template para Data Governance, incluindo:

## 🚀 Como Usar

### Para User Stories
1. Copie o template `user-story-template.md`
2. Preencha as seções obrigatórias
3. Adapte conforme necessário para seu projeto
4. Use em metodologias ágeis (Scrum, Kanban)

### Para Use Cases
1. Copie o template `use-case-template.md`
2. Detalhe todos os fluxos (principal, alternativo, exceção)
3. Inclua regras de negócio relevantes
4. Use em projetos tradicionais ou sistemas complexos

### Para BDD
1. Copie o template `bdd-template.md`
2. Escreva cenários em linguagem Gherkin (português)
3. Mapeie critérios de aceite para cenários
4. Use para comunicação entre equipes e automação de testes
5. Consulte o exemplo prático `bdd-example.md` para referência

### Para Testes
1. **Plano de Teste**: Use `test-plan-template.md` para planejamento geral
2. **Roteiro de Teste**: Use `test-script-template.md` para cenários específicos
3. **Caso de Teste**: Use `test-case-template.md` para testes individuais
4. Execute os testes seguindo a estrutura dos templates
5. Documente resultados e defeitos encontrados

### Para Design e Arquitetura
1. **PRD**: Use `prd-template.md` para requisitos de produto
2. **TRD**: Use `trd-template.md` para referência técnica
3. **FRD**: Use `frd-template.md` para requisitos funcionais
4. **RFC**: Use `rfc-template.md` para propostas de mudança
5. **ADR**: Use `adr-template.md` para decisões arquiteturais
6. **Engineering Guidelines**: Use `engineering-guidelines-template.md` para padrões
7. **System Design**: Use `system-design-template.md` para design de sistema
8. **Architecture Haikai**: Use `architecture-hai-template.md` para documentação concisa
9. **C4 Model**: Use `c4-model-template.md` para documentação em quatro níveis
10. **TRG**: Use `trg-template.md` para revisão técnica
11. **Threat Model**: Use `threat-model-template.md` para análise de segurança
12. **Data Governance**: Use `data-governance-template.md` para governança de dados
13. **High-Level Architecture**: Use `high-level-architecture-template.md` para arquitetura de alto nível
